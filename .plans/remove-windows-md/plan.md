# windows.md 제거

## 무엇을 / 왜

**무엇을**

- `.agents/windows.md`를 삭제한다.
- `AGENTS.md`의 환경별 참고 문서 섹션에서 `.agents/windows.md` 참조를 제거한다.
- 이 문서 삭제가 다른 작업 규칙에 영향을 주지 않는지 최소 범위로 확인한다.

**왜**

- 현재 `windows.md`는 “있으면 자동 적용되는 설정”이 아니라 참고 문서라서, 실제로 에이전트가 매번 정확히 따르지 못하면 토큰 비용만 늘고 실효성이 낮다.
- 사용자 목적은 한글 깨짐 방지 자체보다 “불필요한 문서 읽기 비용 제거”에 가깝다.
- 실제 깨짐 문제는 문서 존재보다, 개별 명령 실행 시 UTF-8을 명시하지 않은 운영 습관 문제다. 문서만 있다고 해결되지 않았다.

## 현재 상태 진단

- `AGENTS.md`에 `.agents/windows.md` 참조가 1곳 있다.
- `.agents/windows.md`는 Windows PowerShell에서 UTF-8 문서를 읽고 GitHub CLI 본문을 다룰 때의 참고 규칙을 적고 있다.
- 그러나 실제 작업 중 plain `Get-Content`가 다시 사용되면 transcript 출력이 깨질 수 있어, 문서 존재만으로 품질이 보장되지 않았다.
- 현재 브랜치에는 이 변경과 무관한 integration 테스트 수정이 진행 중이다. 이번 작업은 문서 삭제만 대상으로 하고, 기존 코드 수정은 건드리지 않는다.

## 해결책

- `AGENTS.md`에서 `.agents/windows.md` 참조 한 줄을 제거한다.
- `.agents/windows.md` 파일을 삭제한다.
- 삭제 후 `.agents` 디렉터리의 나머지 공용 규칙 문서 링크가 자연스럽게 읽히는지만 확인한다.
- 최종 답변에서 “앞으로는 문서 대신 한국어 파일을 읽을 때 명시적 UTF-8 명령을 직접 쓰겠다”는 운영 원칙만 짧게 남긴다.

## 트레이드오프

- 문서를 제거하면 토큰 비용과 불필요한 참조가 줄어든다.
- 대신 PowerShell UTF-8 관련 팀 공용 참고가 사라져, 미래에 같은 문제를 다시 설명해야 할 수 있다.
- 규칙 문서를 줄이면 AGENTS 진입점이 더 단순해진다.
- 반면 환경별 특수 이슈가 다시 생기면 재도입 또는 다른 위치로의 문서화가 필요할 수 있다.

## 대안

1. **`windows.md`를 유지하고 내용을 더 짧게 줄인다**
   - 기각 이유: 핵심 문제는 길이보다 적용 실패다. 짧아져도 “문서가 자동 적용되지 않는다”는 본질은 바뀌지 않는다.
2. **`AGENTS.md`에 한 줄로만 축약한다**
   - 기각 이유: 이 또한 읽기 비용은 줄지만, 결국 운영자가 매번 직접 지켜야 하는 참고 규칙이라는 점은 같다. 사용자는 아예 제거를 원한다.
3. **문서는 두고 내가 더 엄격히 따르기만 한다**
   - 기각 이유: 이미 실제 작업에서 일관되게 적용되지 못했다. 문서 유지의 설득력이 약하다.

## 완료 기준

- [ ] `AGENTS.md`에서 `.agents/windows.md` 참조가 제거된다
- [ ] `.agents/windows.md` 파일이 삭제된다
- [ ] 다른 `.agents` 문서 참조 구조는 그대로 유지된다
- [ ] 이번 변경이 기존 integration 테스트 수정 파일과 충돌하지 않는다
